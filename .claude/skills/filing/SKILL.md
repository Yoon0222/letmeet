# 피넛 운영 문서 정리 (filing)

앱 **코드**가 아니라 **사업/운영 문서**를 다룬다. 대상 루트:
```
C:\Users\SEPC\Documents\P!nut
```

## 대분류 (여기로만 옮긴다)

| 폴더 | 담는 것 |
| --- | --- |
| `00_Inbox` | 새로 받은 자료 임시 보관 (분류 대기) — **여기는 비워가는 게 목표** |
| `01_Store` | App Store / Google Play 심사·메타데이터·스크린샷·앱아이콘·리젝메일 |
| `02_Business` | 사업자등록증, 세금·계좌, 카카오(비즈앱), PG(포트원) 결제·통신판매업 |
| `03_Legal` | 이용약관, 개인정보처리방침, 계약서, 위임장, 임대차계약 |
| `04_Sales` | 코트 입점, 미팅 기록, 제안서, 견적 |
| `05_Marketing` | 랜딩, 이벤트 팝업 자료, 브랜드(로고·컬러), 홍보물·SNS |
| `06_Product` | 로드맵, QA·피드백, 릴리즈 노트, 기획서 |

## 분류 규칙 (키워드 → 폴더)

파일명·확장자·(필요시) 내용을 보고 정한다.

- 사업자등록·세금계산서·홈택스·통신판매·계좌·PG·포트원·카카오비즈 → **02_Business**
- 약관·개인정보·처리방침·계약·contract·임대차·위임 → **03_Legal**
- 스크린샷·screenshot·app-icon·appstore·playstore·심사·리젝·metadata → **01_Store**
- 코트·입점·미팅·제안·견적·proposal·meeting → **04_Sales**
- 랜딩·landing·이벤트·배너·로고·brand·홍보·SNS·포스터 → **05_Marketing**
- 로드맵·roadmap·QA·피드백·릴리즈·release·기획·PRD → **06_Product**
- 애매하면 **00_Inbox 에 그대로 두고 사용자에게 물어본다** (억지로 분류하지 않는다).

## 이름 규칙

- 날짜가 중요한 문서(계약·제출·메일)는 앞에 **`YYYY-MM-DD_`** 를 붙인다. 예: `2026-07-16_사업자등록증.pdf`.
- 공백은 유지해도 되지만, 버전이 있으면 `_v2` 처럼 뒤에 붙인다.
- 이미 잘 정리된 이름은 건드리지 않는다.

## 절차

1. **대상 파악**: 사용자가 파일을 지정하지 않았으면 `00_Inbox`(+요청 시 `Downloads`)를 훑는다.
2. **제안 먼저**: 각 파일에 대해 `파일 → 대상폴더 (+새 이름)` 목록을 만들어 **보여주고**, 애매한 건 표시한다.
3. **확인 후 이동**: 사용자가 승인하면 `Move-Item` 으로 옮긴다.
   - **덮어쓰기 금지**: 대상에 같은 이름이 있으면 옮기지 말고 `_2` 등으로 이름을 바꾸거나 사용자에게 묻는다.
   - **삭제 금지**: 이 스킬은 옮기기만 한다. 지우는 건 사용자가 직접.
   - 대상 대분류 아래 소분류 폴더가 필요하면(예: `01_Store\screenshots`) 만들어도 된다.
4. **마무리**: 무엇을 어디로 옮겼는지 요약. `00_Inbox` 가 비었는지 확인.
