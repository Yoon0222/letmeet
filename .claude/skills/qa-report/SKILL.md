# QA 리포트 (노션 누적 + 정리)

QA 결과는 **노션 한 페이지에 계속 쌓고**, 나중에 한 번에 정리한다.

## 대상 노션 페이지
- 제목: **피클 — QA 리포트 & 이슈**
- ID: `39012248-242a-81d6-b54c-d8252620012b`
- URL: https://app.notion.com/p/39012248242a81d6b54cd8252620012b

(페이지가 없거나 못 찾으면 `notion-search "피클 QA 리포트"` 로 찾고, 없으면 워크스페이스 최상위에 같은 제목으로 새로 만든다.)

## A) 새 QA 결과 누적 (qa-pickle 실행 후마다)
`qa-pickle` 리포트를 받으면 아래를 한다:
1. `notion-fetch` 로 페이지를 읽어 같은 기능·날짜 엔트리가 이미 있는지 확인(중복 방지).
2. `notion-update-page` (command: `insert_content`, position: `{"type":"start"}`) 로 **인트로 blockquote 바로 아래(최상단)** 에 새 엔트리를 추가 → 최신이 위로 온다.
3. 엔트리 형식(기존 페이지와 동일하게):
```markdown
## <기능명> — <YYYY-MM-DD> · <PASS | FAIL | PARTIAL>
<한 줄 요약: 정적 결과 · 동작확인 · 수용기준 n/m>

**미해결 지적**
- [ ] [High|Med|Low] `파일:라인` — 무엇이 왜 / 재현·영향
```
- 미해결 지적이 없으면 "지적 없음." 한 줄만.
- 날짜는 세션 컨텍스트의 currentDate(절대날짜).

## B) 전체 정리 (요청 시: "QA 정리/모아줘")
1. `notion-fetch` 로 전체를 읽는다.
2. 페이지 **최상단**에 `## 📊 요약 (YYYY-MM-DD 기준)` 섹션을 만들거나 갱신:
   - 미해결 `- [ ]` 개수를 심각도별 집계 (High/Med/Low).
   - 심각도 높은 순으로 미해결 항목을 한데 모아 나열 (기능·파일:라인 포함).
3. 코드에서 **이미 해결된 지적**은 `- [ ]` → `- [x]` 로 바꾼다 (update_content).
4. 필요하면 오래된 PASS 엔트리는 접거나 유지 — 히스토리는 지우지 않는다.

## 원칙
- 노션에 쓰는 건 사용자 워크스페이스 변경이니, **누적(append)** 은 자유롭게 하되 **대규모 재작성/삭제 전엔 확인**한다.
- 심각도 기준: High=반드시 수정 · Med=수정 권장 · Low=선택.
- 로컬 `docs/WORKLOG.md` 의 결정 기록과 역할이 다르다 — WORKLOG는 "무엇을 왜", QA 페이지는 "검증 결과·이슈".
