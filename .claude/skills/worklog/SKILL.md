# 워크로그 (피클 작업 일지)

`docs/WORKLOG.md` 하나에 **핵심만** 계속 쌓는다. 장황한 대화 전문이 아니라, 나중에 봐도 맥락이 복원되는 **결정 기록(decision log)** 이다.

## 언제 갱신하나
- 기능/화면/DB 변경을 **완료**했을 때
- 방향을 바꾸는 **결정**을 내렸을 때 (예: 탭 의미 변경, 스택 선택)
- 사용자가 "기록/로그 정리해줘" 라고 할 때
- 세션을 마무리할 때

## 절차
1. `docs/WORKLOG.md` 를 읽는다. (없으면 아래 구조로 생성)
2. **직전 엔트리 이후** 일어난 일만 요약해 새 하위 엔트리(`### 제목`)를 만든다.
   - **오늘 날짜 섹션(`## YYYY-MM-DD`)이 이미 있으면** 그 섹션 안 최상단에 하위 엔트리를 추가한다. (날짜 헤더를 새로 찍지 말 것 — 같은 날짜 중복 금지)
   - **없으면** 엔트리 영역 최상단에 `## 오늘날짜` 섹션을 새로 만들고 그 아래 첫 엔트리를 넣는다.
3. 상단 "## 열린 항목" 체크리스트를 **최신화**한다 (끝난 건 [x], 새로 생긴 건 추가).
4. 날짜는 상대표현 금지 — 절대날짜(YYYY-MM-DD)로. (오늘 날짜는 세션 컨텍스트의 currentDate 사용)

## 노션 기능 현황도 함께 갱신 (필수)

기능을 **완성**했으면 `docs/WORKLOG.md` 뿐 아니라 **노션 기능 페이지**에도 항목을 추가한다.

- 대상: **"피넛 — 구현 기능 전체 정리"** (page id `39f12248-242a-81cc-a677-dc8d30cac044`)
- 맨 앞 **"🚦 출시 상태 한눈에"** 표에 행을 추가하고, 상태는 **🔵 미배포 / 버전 "다음"** 으로 넣는다 (아직 스토어에 안 나갔으므로).
- 아래 도메인별 상세 목록(계정·번개·대회·코트 등)에도 한 줄 추가한다.
- 상태 표기: 🟢 공개 출시 · 🟡 테스트 배포 · 🟠 심사 대기 · 🔵 개발 완료·미배포 · ⚪ 보류
- 배포되면 `release` 스킬이 🔵 → 🟡/🟢 로 옮기고 릴리스 노트에 옮겨 적는다. **여기서는 🔵까지만.**

## DB 마이그레이션 실행 현황도 갱신 (필수)

마이그레이션(`supabase/migrations/NNNN_*.sql`)을 **새로 만들면** 노션에 행을 추가하고, **사용자가 실행했다고 하면** 표시를 바꾼다.

- 대상: **"피넛 — DB 마이그레이션 실행 현황"** (page id `3a312248-242a-8119-bb2f-db6ef6910d61`)
- 새로 만들면 → "미실행 목록" 표에 `dev ⬜ / prod ⬜` 로 행 추가
- 실행했다고 하면 → 해당 칸 ✅ 로 변경. **가능하면 REST 조회로 실제 반영을 확인한 뒤** 바꾼다(착각 방지):
  ```
  {SUPABASE_URL}/rest/v1/{테이블}?select={새컬럼}&limit=1   → 200이면 반영됨
  ```
  dev 키는 `.env`, prod 키는 `eas.json` 의 `build.production.env` 에 있다(anon 키라 조회 안전).
- 양쪽 다 ✅ 되면 "실행 완료" 섹션으로 옮긴다.
- **배포 전 체크**: 스토어 빌드나 Vercel 배포 전에 prod 칸에 ⬜ 가 없는지 확인 — 있으면 먼저 실행하도록 알린다.

## 파일 구조
```markdown
# 피클 — 작업 일지 (WORKLOG)

## 열린 항목
- [ ] 아직 안 끝난 할 일 (누가/무엇)
- [x] 끝난 건 체크

---

## YYYY-MM-DD        ← 하루당 헤더 하나. 최신 날짜가 위로.

### <한 줄 제목>      ← 그날의 작업 하나. 같은 날 새 작업은 이 섹션 안 위쪽에 추가.
- **결정**: 무엇을 왜 그렇게 정했는지 (1~3줄)
- **만든 것**: 파일/기능 (핵심만, 경로 포함)
- **메모/주의**: 함정, 후속 필요 (선택)

### <또 다른 제목>
- ...
```

## 작성 규칙
- 엔트리는 **6줄 이내**를 목표로. 코드 diff 를 붙이지 말고 "무엇을/왜"만.
- 결정의 **이유**를 꼭 남긴다 (나중에 "왜 이렇게 했지?" 를 없앤다).
- 파일 언급은 경로로 (`src/app/(tabs)/clubs.tsx`).
- 완료된 열린 항목은 지우지 말고 [x] 로 남겨 히스토리를 유지.
- 노션 연동이 켜지면 이 파일을 노션 페이지에도 미러링할 수 있다(마크다운 그대로 붙여넣기 가능).
